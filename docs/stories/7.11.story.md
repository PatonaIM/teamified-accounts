# Story 7.11: Configuration Management UI (Tax Years, Exchange Rates, Region Settings)

## Status
Ready for Review

## Story
**As a** system administrator,
**I want** user-friendly frontend interfaces for managing tax years, exchange rates, and region-specific configurations,
**so that** I can configure multi-region payroll settings efficiently through the web interface without requiring database access or technical knowledge.

## Acceptance Criteria
1. **Tax Years Management UI:** Complete CRUD interface for managing country-specific tax years with validation
2. **Exchange Rates Management UI:** Complete CRUD interface for managing currency exchange rates with historical support
3. **Region Configurations UI:** Complete CRUD interface for managing country-specific configuration settings
4. **Material-UI Integration:** All components follow Material-UI 3 Expressive Design System patterns from Stories 7.1 and 7.2
5. **Country Context Integration:** All components integrate with existing CountryContext from Story 7.1
6. **API Integration:** Use existing backend APIs from Story 7.1 (no backend changes required)
7. **Validation:** Comprehensive client-side validation with proper error handling and user feedback
8. **Responsive Design:** Mobile-friendly interfaces that work on all device sizes
9. **Code Cleanup:** Remove duplicate Payroll Periods tab from PayrollConfigurationPage (functionality moved to Payroll Administration in Story 7.8)
10. **Testing:** Comprehensive frontend component testing following existing patterns

```typescript
interface TaxYear {
```

```typescript
const { selectedCountry, loading, error } = useCountry();
// selectedCountry: Country | null - contains code, name, currency, etc.
```

**Pattern:**
- All API calls should use `selectedCountry.code` as the country parameter
- Show loading state while `loading === true`
- Show error message if `error !== null`
- Disable forms/actions if `selectedCountry === null`

### Validation Rules

**Tax Year Validation**
[Source: src/payroll/dto/tax-year.dto.ts]
- Year must be >= current year
- Start date is required and must be a valid date
- End date is required and must be a valid date
- End date must be after start date
- Date range should be approximately 12 months (warning, not error)

**Exchange Rate Validation**
[Source: src/payroll/dto/exchange-rate.dto.ts]
- From currency and to currency are required
- From currency must not equal to currency
- Rate must be > 0
- Effective date is required and must be a valid date
- Effective date cannot be in the past (warning, not strict error)

**Region Configuration Validation**
[Source: src/payroll/dto/region-configuration.dto.ts]
- Config key is required, max 100 chars, alphanumeric + underscores only
- Config key must be unique within the country
- Config name is required, max 200 chars
- Config value is required and must match value type
- Value type is required (string, number, boolean, json)
- For JSON value type, validate JSON syntax before submission
- Description is optional, max 500 chars
